# SEID 2364 Case Study Assignment Execution Protocol
## Standard for Framed and Student-Selected Cases

## Purpose
This protocol standardizes how case-study assignments are designed, assigned, and evaluated in SEID 2364.

It supports two modes:
- instructor-framed case studies;
- student-selected case studies.

It also defines how much information belongs in lite case packets versus fuller instructor packets.

## Core Design Principle
Case packets should provide enough shared context for fair evaluation, but not so much analysis that students are pre-led to conclusions.

## Assignment Modes

### Mode A: Instructor-Framed Case
Use when the class needs a shared reference system or comparative baseline.

Required elements:
- common case packet for all students;
- fixed minimum evidence set;
- shared prompt and deliverable structure;
- student-owned analysis outputs.

### Mode B: Student-Selected Case
Use when the learning objective is transfer, synthesis, or independent inquiry.

Required elements:
- approved case selection process;
- evidence quality threshold;
- common analysis template (pathway, stakeholders, risks, frameworks, recommendation);
- comparability rubric across different case topics.

### Mode C: Hybrid
Use when students may choose either a framed case or an approved self-selected case.

Required elements:
- same deliverable structure for both paths;
- same rubric dimensions;
- explicit evidence standards for self-selected cases.

## Required Student Analysis Outputs (All Modes)
Every case assignment must require students to produce, not just summarize, the following:

1. Mediation pathway map:
- trace source -> transformation -> decision/output -> affected destinations;
- mark control and observability points.

2. Stakeholder model:
- identify direct, economic/institutional, governance/oversight, and design/accountability actors;
- justify inclusion and exclusion decisions.

3. Bidirectional risk model:
- harms if implemented/intervened;
- harms if not implemented/intervened;
- tradeoff harms present in both directions.

4. Framework comparison:
- apply at least two ethical/professional frameworks;
- identify agreement, divergence, and unresolved tensions.

5. Governance recommendation:
- propose policy/process/technical controls;
- include transparency, appeal, and accountability mechanisms.

## Case Packet Depth Standard

### Lite Case Packet (Student-Facing)
Use for final projects, major assessments, and independent analysis.

Must include:
- short case brief;
- core facts;
- evidence packet links;
- tasks and required deliverables;
- constraints.

Must not include:
- pre-solved stakeholder map;
- pre-solved harm map;
- fully interpreted final ethical conclusion.

### Standardized Case Packet (Teaching/Scaffold)
Use for in-class learning, guided workshops, and early skill-building.

May include:
- narrative framing;
- example stakeholder groupings;
- example bidirectional risk framing;
- richer discussion prompts.

### Instructor Packet (Support)
Use for teaching notes and facilitation.

May include:
- anticipated misconceptions;
- discussion pivots;
- model solution outlines;
- assessment signals.

## Decision Rule for "How Lite is Lite"
Before finalizing a lite case packet, run this check:

1. Could a student submit a high-scoring paper by paraphrasing the packet?
- If yes, the packet is too heavy and must be reduced.

2. Does the packet still provide enough factual grounding for fair analysis?
- If no, add neutral context and evidence links.

3. Are key analysis products still student-generated?
- If no, remove embedded conclusions.

## Minimum Evidence Standard

### For Instructor-Framed Cases
Students must cite:
- primary case source;
- required frameworks/codes specified in assignment;
- at least one supporting source when assigned.

### For Student-Selected Cases
Students must submit:
- case justification statement;
- minimum 3 credible sources (at least 1 primary or official source);
- source quality note explaining reliability and limitations.

## Assignment Build Template
Each case assignment should include these sections:

1. Framing and objective
2. Case mode (A, B, or C)
3. Required case/evidence materials
4. Required student outputs
5. Deliverables and format
6. Evaluation criteria
7. Quick checklist

## Evaluation Dimensions (Common Rubric Spine)
Use these dimensions in all case assignments:
- pathway clarity and completeness;
- stakeholder rigor and justification;
- risk/harm analysis quality (including bidirectional reasoning);
- framework application depth;
- governance recommendation quality;
- use of evidence and source integrity;
- clarity and organization.
