## Part C Operating Constraints (Read Before Worksheet C)

Worksheet C is a reconstruction drill for method fluency, not a continuation of Assignment 5 narrative.

- Use a simple, non-ethically loaded practice case only in Worksheet C.
- Do not map your Assignment 6 system in Worksheet C.
- In Assignment 6, transfer this method to your selected course system.
- Use one and only one mediation pathway per Part C sheet.
- Duplicate Part C for each additional pathway.

Default classroom mode (simple exercise):

- Keep Part C lightweight and drawing-first.
- Require one explicit SVD line.
- Define the pathway endpoints first: ultimate source, ultimate destination, source/destination sensorium, and source/destination space type.
- Use the transition table to record each step from start state to end state.
- For each step, note the transformation/refactoring that occurs.
- Use the diagram section to redraw the same pathway visually.
- Use short concrete wording; avoid long analytical prose.

Extended analysis mode (optional):

Use this mode only when students are ready for denser pathway analysis.

Required pathway construction rules:

- Build at least 3 pathways across the full exercise sequence: 1 simple, 1 chained (minimum 3 explicit SVD steps), 1 branching or recursive.
- Write the full pathway in explicit SVD form, then break transitions into smaller steps where needed.
- The transition list should capture movement between spaces/sensorium plus the transformation or refactoring that occurs.

```text
Source -> Vector -> Destination
Transformation: [what changes between source and destination]
Access: [who/what can perceive, modify, or block this step]
```

- For chained pathways, write each step separately (do not compress).
- For branching or recursive pathways, mark the branch/recursion point explicitly.

Modality and representation rules (required):

- For each pathway step, identify the physical sensorium channel(s) involved where relevant (for example: visual, audible, textual, tactile, gestural).
- Identify how the virtual layer packages/transmits the equivalent representation (for example: bitmap/image, audio file/stream, text token sequence, table row, vector embedding).
- Mark whether the information state at that step is primarily raw signal, structured data, or conceptually embedded representation.
- Note the transformation from one representation state to another when it changes.

Storage and trigger edge-cases (important):

- A destination may be storage/queue/archive rather than an active perceiving actor.
- In those cases, mark time mode as delayed or indeterminate until retrieval/activation occurs.
- A source may be trigger-only (for example: sends/transfers a file without conceptual access to contents).
- Do not assume conceptual awareness when the source only initiates transmission.

Point-of-origin and distance rules:

- Declare one primary actor perspective before tracing pathways.
- Rewrite at least one pathway from a different point of origin.
- Note what changes when perspective shifts (source, vector visibility, distance, access, responsibility).
- Where useful, annotate pathways with:

```text
Distance:
- Space: personal/local, remote, unreachable
- Time: real-time, delayed, recorded, inaccessible
```

Symmetry and break rules:

- Identify one pathway where cognitive/computational symmetry holds.
- Identify one pathway where symmetry breaks.
- For each, provide:

```text
Symmetry Condition:
Break Condition:
Why:
```

Precision rule (anti-vagueness):

- Do not use compressed phrases such as "the system processes," "data is analyzed," or "information is sent" without naming actor, mechanism, and transformation.

Compression audit requirement:

- Identify one likely compressed pathway step.
- Expand it into at least 2-3 additional explicit SVD steps with transformation and access lines.
